# PR Skill

## 注意点

- test や lint などのチェックはこのスキルでは絶対に実行してはならない
- 作業効率化のためのスキルであるため、最短で目的を達成することを最優先にする

## 手順

- 現在のブランチを確認する
  - main ブランチに居る場合は、新しいブランチを作成する
  - main ブランチ以外に居る場合は、そのまま作業を続ける
- ステージされている変更を確認する
  - ステージされていない変更がある場合はステージする
- コミットする
  - コミットメッセージは深く考えずに作成する
- プッシュする
- 既にこのブランチの GitHub PR が存在しているか確認する
  - 既に存在する場合はそこで終了する
  - まだ　PR が存在しない場合は gh コマンドを使用して、GitHub に PR を作成する
    - PR のタイトルや説明は大雑把でよい。深く考えてはならない
