- タスクに対してコマンドを実行できる機能を追加する
- 任意のコマンドを実行できる
- 主な用途はAIコーディングエージェントの起動を想定
  - `claude` や `codex` など
  - ユーザーによって前処理として git worktree を切ったりコンテナ環境を立ち上げたりといった操作が必要だと思われるので、任意のコマンドを入力できるようにする
- ローカルで実行することが前提のツールであるため、セキュリティについては利用者の自己責任とする
- タスクの内容（＝MARKDOWN BODY）を引数（例えばAIへのプロンプト）として渡せるようにする
  - チェックボックスで、引数として渡すかを ON/OFF 切り替え
  - 引数なのか標準入力なのかも選べるようにする
- 複数のコマンドを実行できるようにする。二つ目以降のコマンドは、前のコマンドの標準出力を標準入力として受け取って実行する
- 特殊な変数として、タスクのタイトルを `$TASK_TITLE`、ファイルパスを `$TASK_FILEPATH`、タスク本文を `$TASK_BODY` として利用可能にする
- 特殊なコマンドとして GOTO を使えるようにする。git worktree を使う場合にはカレントディレクトリの移動が必要だが、個別のスクリプト実行ではカレントディレクトリ移動が色々と面倒なため

## 使用例

以下のような複数コマンドを想定

1. `git worktree add -b $TASK_TITLE $TASK_TITLE`
2. `GOTO $TASK_TITLE`
3. `./create-prompt-then-print.sh $TASK_BODY`
4. `claude`
